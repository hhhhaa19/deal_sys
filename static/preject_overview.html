Project Architecture Overview

Frontend: The interface uses HTML, CSS, JavaScript, and jQuery to manage dynamic content and asynchronous requests.

Backend: Development is based on Python's Flask framework, which supports simplified backend structure and rapid development.

Project Structuring: The project experienced redundancy issues with multiple teams fetching the same BTC data from three different sources, leading to a lack of clear structure and layering. This has impacted code maintainability and readability, suggesting a need for a detailed initial task assignment and a microservices approach for clearer service boundaries and easier future expansions.

Thread Safety: Flask's built-in thread pool elegantly handles thread safety. Transactions operate in separate threads created via threading.Thread to avoid blocking the thread pool, with threading.Event objects managing start and stop signals for trading operations. A global variable trading_pair is protected by a trading_pair_lock to ensure thread-safe operations.

Database Management: The database handling is secure against SQL injections by validating queries server-side. However, the lack of a connection pool results in high overhead from frequent open/close connections. Introducing a database connection pool and handling potential concurrency issues like dirty reads or phantom reads could improve performance and reliability.

Automation and Service Availability: Service availability is robust due to unified exception handling and well-maintained logs, which simplify maintenance and debugging. Future enhancements could include SMS notifications to further automate and improve service responsiveness.
